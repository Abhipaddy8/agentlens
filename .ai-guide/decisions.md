# Decisions Log — AgentLens

### 2026-03-10 Decision: Use Node.js for proxy / Reason: OpenAI SDK is Node-native, Lambda has first-class Node support, matches team skills
### 2026-03-10 Decision: DynamoDB over Postgres / Reason: Serverless, scales with Lambda, sub-100ms reads, no connection pooling needed, pay-per-request
### 2026-03-10 Decision: React for dashboard / Reason: Fastest path to 5 screens, massive ecosystem, deploys to S3+CloudFront trivially
### 2026-03-10 Decision: CloudFormation over Terraform / Reason: Native AWS, customers click one link in AWS console, no external tooling required
### 2026-03-10 Decision: Monorepo structure (proxy/ dashboard/ infra/ demo/) / Reason: Single repo for all components, simpler CI, one git history
### 2026-03-10 Decision: OpenAI-compatible endpoint only (v1) / Reason: 80%+ of enterprise LLM spend is OpenAI. Anthropic/others come after first customer.
### 2026-03-11 Decision: Fork Chatbot UI (mckaywrigley) for Agent Studio / Reason: Thin scaffold, Next.js + TS + Tailwind matches stack, easy to gut. Lobe Chat considered but too heavy — would spend more time understanding its plugin architecture than building. Chatbot UI gives clean chat rendering + streaming, we build the conversation controller + inline components on top.
### 2026-03-11 Decision: Three parallel build agents (F/B/I) per mission / Reason: Frontend, Backend, and Infra touch different files. Isolated worktrees, no merge conflicts. Cuts mission time by 2-3x on heavy missions (M16, M25, M26).
### 2026-03-11 Decision: TechStack Tetris architecture patterns absorbed into Agent Studio / Reason: Pipeline blocks (visible in chat), crash recovery (3-5 pattern: checkpoint + retry + reconciliation), safe rollout (4-5 pattern: shadow test + auto-rollback), query routing (3-2 pattern: classifier → tool selector → data source), observability auto-wiring (4-2 pattern). These are production-grade patterns proven across 30 missions. Every agent built through Studio inherits them by default.
### 2026-03-11 Decision: Inline games (Tetris/Snake/Pac-Man) during background builds / Reason: Biggest UX problem with autonomous build systems is the wait. Users alt-tab, forget, lose momentum. Games keep them in the tab. Open source React/Canvas components, game receives play/pause signals from build runner.
### 2026-03-11 Decision: agentlens-agent.json as standard output schema / Reason: Every agent built by NPC Guide must conform to one schema. Enables: one-click deploy (Lambda knows the format), safe rollout (compare old vs new schema), guardrails validation (build fails if invalid). Schema includes: entry point, triggers, integrations, schedule, memory config, human-in-the-loop rules, checkpoint config.
### 2026-03-11 Decision: All built agents auto-wire through AgentLens proxy / Reason: No agent deploys without observability. Every LLM call goes through the proxy — cost tracking, caching, routing, kill switches all work automatically. This is the platform play: Agent Studio builds it, the proxy monitors it.
